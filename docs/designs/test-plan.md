## Affected Pages/Routes
- `/` (main page) — new file explorer panel, modified office canvas (badges), modified spawn form (repo field + folder picker)

## Key Interactions to Verify
- Spawn agent with repo path → worktree created, agent appears in all 3 panels
- Click branch in explorer → three-way sync (terminal, office, explorer all switch)
- Click modified file → inline diff expands with colored lines
- Cmd+E → toggle explorer panel visibility
- Cmd+1..9 → switch between agents
- Kill agent → worktree cleanup (keeps if dirty, removes if clean)

## Edge Cases
- Spawn with invalid/non-existent repo path → clear error message
- Spawn with repo that has no commits → clear error message
- Spawn with duplicate agent name → clear error message
- Kill agent with uncommitted changes → worktree preserved, user notified
- Corrupted config.json on startup → graceful degradation (empty config, no overwrite)
- Very large repo → git status timeout after 5s, scan skipped
- Narrow viewport (<900px) → explorer auto-collapses
- Two agents modify same file → conflict warning appears in explorer + office

## Critical Paths
- Path validation: repoPath from WebSocket must be absolute, exist, resolve, be git repo → security boundary
- Worktree lifecycle: create on spawn, cleanup on kill, prune on startup → no orphaned worktrees
- State-triggered scanning: WORKING→IDLE → git status → tree broadcast → conflict check → full pipeline
- Three-way sync: any panel click → switchToSession() → all panels update
